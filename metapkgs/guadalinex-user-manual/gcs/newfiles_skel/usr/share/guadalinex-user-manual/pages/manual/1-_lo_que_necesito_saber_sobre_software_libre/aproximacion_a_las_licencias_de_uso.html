Aproximación a las licencias de uso

Mucho se ha hablado y se ha escrito sobre las licencias de software, ya que es un tema complicado, controvertido y lleno de sutilezas legales, que hacen que su comprensión a veces sea difícil y lenta.

Las licencias son partes integrantes del software libre, ya que depende de ellas proteger este software para que siga siendo libre.

Como no queremos liarte en exceso y el tema es extenso, vamos a explicarte los conceptos básicos de las licencias, y una vez que los entiendas de forma general, podrás ir ampliando tus conocimientos.

Una licencia de software es un contrato entre el autor, el titular de los derechos de explotación, o el distribuidor y el usuario consumidor del programa informático. Por decirlo de otra manera, es una autorización formal que el autor del software da a un interesado o usuario para poder explotar de forma legal ese software.

En el caso de las licencias de software libre, vamos a aclararte primero lo que es el Derecho de autor y el Copyright que utiliza el software privativo, para que veas las diferencias y lo puedas entender mejor.

El Derecho de autor se basa en una idea sobre la identidad personal que reconoce que una obra ha sido creada por una persona concreta. Y el Copyright se limita estrictamente a la obra, aspectos relacionados con su reproducción, modificación, publicación, venta y distribución.

En contraposición a los derechos de autor y al copyright, surge en el entorno del software libre el Copyleft, licencia libre que significaba una forma de especificar que cualquier persona que redistribuya el software, con o sin cambios, no puede restringir a nadie la libertad de copiarlo, redistribuirlo o cambiarlo. De esta manera, el Copyleft garantiza las libertades de los usuarios.

Mientras que los desarrolladores de software privativo emplean el copyright para limitar y coartar la libertad de los usuarios, los desarrolladores de software libre utilizan este copyright para garantizar la libertad de los usuarios respecto al uso del producto, sólo tendrás que citar o reconocer quien ha sido el autor original de ese software.

Si hablamos de las licencias relacionadas con el software libre, la mayoría pertenecen a grupos de licencias promovidas por dos organismos que se encargan de examinarlas y comprobar si una licencia cumple todos los requisitos que impone dicho organismo para que sea de software libre.

Estas licencias forman grupos según sean promovidas y aprobadas por:

  • Free Software Foundation o FSF (Fundación de Software Libre).
  • Open Source Initiative (OSI).
Aparte de los programas, software o aplicaciones, también los contenidos, textos o documentos reciben este tipo de licencias.Una de las más utilizadas es la Creative Commons.
